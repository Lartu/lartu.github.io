Obsidian Organization

This document details how I plan to organize the documents within this knowledge base so they are accessible transitively from the root document, and so they maintain a cohesive style.

Document Organization
...

  • Documents should each be contained in a subfolder within the knowledge base root, with the exception of the Index document, that should serve as the entry point for the Memex.

Design Choices
...

  • Section titles should use a size equivalent to ##. A line break is left before a section if there was a section before it. No line breaks should be left after it.
  • Subtitles within documents should use a size equivalent to ### or less. No line breaks should be added before nor after subtitles.
